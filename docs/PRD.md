## 1. 프로젝트 개요

### 1.1 프로젝트명
**Shorts Auto Next**

### 1.2 한 줄 요약
유튜브 쇼츠 영상이 끝나면 자동으로 다음 영상이 재생되는 Chrome 확장 프로그램

### 1.3 프로젝트 동기
- 유튜브 쇼츠는 영상이 끝나면 자동으로 반복 재생됨
- 다음 영상을 보려면 스와이프하거나 클릭해야 함
- 이는 멀티태스킹 중이거나 오디오로만 청취하는 사용자에게 불편함

### 1.4 솔루션
영상 재생이 끝나는 시점을 감지하여 자동으로 다음 쇼츠로 이동

## 2. 문제 정의

### 2.1 현재 상황 (As-Is)
```
사용자가 쇼츠 시청 → 영상 끝남 → 자동 반복 재생 → 수동으로 다음 영상 넘김 필요
                                                    ↑
                                              [불편한 포인트]
```

### 2.2 목표 상황 (To-Be)
```
사용자가 쇼츠 시청 → 영상 끝남 → 자동으로 다음 영상 재생 → 계속 시청
                              ↑
                        [Shorts Auto Next가 해결]
```

### 2.3 핵심 문제
| 문제 | 영향 | 심각도 |
|------|------|--------|
| 영상마다 수동으로 넘겨야 함 | 시청 흐름 끊김 | ⭐⭐⭐ |
| 화면을 안 보면 넘기기 어려움 | 오디오 청취자 불편 | ⭐⭐⭐ |
| 손을 쓸 수 없는 상황에서 불편 | 멀티태스킹 방해 | ⭐⭐ |

## 3. 타겟 사용자

### 3.1 주요 페르소나

#### 페르소나 A: "오디오 리스너" 민수 (28세, 직장인)
- **행동**: 유튜브 쇼츠를 틀어놓고 소리로만 들으며 다른 작업
- **상황**: 재택근무 중, 요리하면서, 청소하면서
- **니즈**: 손을 쓰지 않고 계속 새로운 영상 듣기
- **불만**: "영상 끝날 때마다 와서 넘겨야 해서 귀찮아"

#### 페르소나 B: "PC 헤비유저" 지영 (24세, 대학생)
- **행동**: PC에서 유튜브 쇼츠를 브라우저로 시청
- **상황**: 과제하면서, 게임 대기 중, 밥 먹으면서
- **니즈**: 마우스/키보드 조작 최소화
- **불만**: "모바일처럼 스와이프도 안 되고 클릭해야 해서 불편해"

### 3.2 사용 환경
| 환경 | 비중 | 특징 |
|------|------|------|
| 데스크톱 PC | 60% | 대화면, 멀티태스킹 |
| 노트북 | 40% | 이동 중 사용 |

### 3.3 사용 시나리오
1. **시나리오 1**: 재택근무 중 BGM처럼 쇼츠 틀어놓기
2. **시나리오 2**: 식사하면서 손 안 쓰고 쇼츠 보기
3. **시나리오 3**: 게임 로딩/대기 중 쇼츠 자동 재생

## 4. 핵심 기능

### 4.1 기능 우선순위

| 우선순위 | 기능 | 설명 | MVP |
|----------|------|------|-----|
| P0 | 자동 다음 재생 | 쇼츠 영상 끝나면 자동으로 다음 영상 | ✅ |
| P0 | ON/OFF 토글 | 자동재생 기능 켜기/끄기 | ✅ |
| P1 | 설정 저장 | ON/OFF 상태 브라우저에 저장 | ✅ |
| P2 | 관심없음 버튼 | 현재 영상 스킵 + 유튜브에 피드백 | ❌ |
| P3 | 채널 스킵 | 특정 채널 자동 스킵 | ❌ |

### 4.2 MVP 기능 상세

#### 4.2.1 자동 다음 재생
```
[트리거 조건]
- YouTube Shorts 페이지 (/shorts/* URL)
- 영상 재생 시간이 끝에 도달
- 자동재생 기능이 ON 상태

[동작]
- 다음 영상으로 스크롤 또는 네비게이션 실행
- 유튜브의 기본 "다음 영상" 버튼 클릭 시뮬레이션
```

#### 4.2.2 ON/OFF 토글
```
[UI 위치]
- 쇼츠 화면 우측 하단 플로팅 버튼

[상태 표시]
- ON: 아이콘 활성화 + "자동재생 켜짐" 툴팁
- OFF: 아이콘 비활성화 + "자동재생 꺼짐" 툴팁

[인터랙션]
- 클릭 시 토글
- 상태 변경 시 부드러운 피드백 애니메이션
```

#### 4.2.3 설정 저장
```
[저장 항목]
- autoPlayEnabled: boolean (기본값: true)

[저장 위치]
- Chrome Storage Sync API
- 동일 계정 브라우저 간 설정 동기화

[로드 시점]
- 확장 프로그램 초기화 시
- YouTube Shorts 페이지 진입 시
```

### 4.3 향후 기능 (Post-MVP)

#### Phase 2: 관심없음 기능
- 현재 영상 즉시 스킵
- 유튜브 "관심 없음" API 호출 (알고리즘 반영)

#### Phase 3: 채널 관리
- 특정 채널 블랙리스트/화이트리스트
- 블랙리스트 채널 자동 스킵

#### Phase 4: 고급 설정
- 자동 넘김 딜레이 설정 (0초 ~ 5초)
- 키보드 단축키 지정
- 시청 통계 대시보드

### 5.5 영상 끝 감지 방법

```javascript
// 방법 1: timeupdate 이벤트
video.addEventListener('timeupdate', () => {
  if (video.currentTime >= video.duration - 0.5) {
    triggerNextVideo();
  }
});

// 방법 2: ended 이벤트 (쇼츠는 루프라 발생 안 할 수 있음)
video.addEventListener('ended', () => {
  triggerNextVideo();
});

// 방법 3: 시간 비교 (폴링)
setInterval(() => {
  if (video.currentTime >= video.duration - 0.3) {
    triggerNextVideo();
  }
}, 500);
```
